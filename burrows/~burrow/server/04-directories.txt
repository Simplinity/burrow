# Directories & Config

A burrow is a directory of files. A directory can have a config file. That's the whole data model.

---

Every directory can contain a .burrow file with up to five keys:

  description = ITAD, web craft, and too many opinions about typography
  accent = #e06030
  title = My Fancy Name
  sort = modified-desc
  pin = important-post.txt readme.txt

description: shown next to the directory name in listings. Your elevator pitch in one line.

accent: a hex color that overrides the default teal. Your burrow, your color. Validated server-side because we don't trust your CSS instincts. (Just kidding. Mostly.)

title: overrides the directory name in headings. Because phlog/ deserves better.

sort: controls listing order. Options are name-asc (default), name-desc, modified-desc (newest first, perfect for phlogs), and modified-asc.

pin: space-separated filenames that get pinned to the top, in the order you specify. Your greatest hits, always above the fold.

---

The config format is deliberately flat. No nesting. No YAML. No TOML sections. No JSON braces. One key = one value per line. If you can read a grocery list, you can read a .burrow file.

There are exactly ten config keys in the entire system: five for the server, five per directory. The full configuration of a Burrow deployment fits in a tweet. We consider this a feature.
